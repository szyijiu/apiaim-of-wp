## 一、技术栈

| 层级 | 技术选型 | 说明 |

## 四、数据库设计 (MySQL)

### 4.1 用户相关表

```sql
-- 用户表
CREATE TABLE `user` (
  `id`            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `email`         VARCHAR(255) NOT NULL UNIQUE,
  `password_hash` VARCHAR(255) NOT NULL,
  `nickname`      VARCHAR(100) DEFAULT '',
  `avatar`        VARCHAR(500) DEFAULT '',
  `phone`         VARCHAR(20) DEFAULT '',
  `wechat_openid` VARCHAR(100) DEFAULT '' UNIQUE,
  `role`          ENUM('admin','merchant') DEFAULT 'merchant',
  `status`        TINYINT DEFAULT 1 COMMENT '1=active 0=disabled',
  `created_at`    DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at`    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 角色表 (RBAC)
CREATE TABLE `role` (
  `id`          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `name`        VARCHAR(50) NOT NULL UNIQUE COMMENT '管理组/名商组',
  `permissions` JSON COMMENT '权限列表',
  `created_at`  DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 用户-角色关联
CREATE TABLE `user_role` (
  `user_id` INT UNSIGNED NOT NULL,
  `role_id` INT UNSIGNED NOT NULL,
  PRIMARY KEY (`user_id`, `role_id`),
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`role_id`) REFERENCES `role`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.2 域名相关表

```sql
-- 域名主表
CREATE TABLE `domain` (
  `id`                INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `name`              VARCHAR(255) NOT NULL COMMENT '域名名称',
  `meaning`           TEXT COMMENT '域名含义解读',
  `suitable_industry` VARCHAR(500) COMMENT '适合行业',
  `price`             DECIMAL(12,2) DEFAULT 0.00 COMMENT '挂牌价格',
  `status`            ENUM('negotiable','locked','urgent') DEFAULT 'negotiable' COMMENT '出售状态',
  `registered_at`     DATE DEFAULT NULL COMMENT '注册时间',
  `expiry_years`      INT DEFAULT NULL COMMENT '到期年限',
  `heat`              INT DEFAULT 0 COMMENT '热度值 0-999999',
  `visits`            INT DEFAULT 0 COMMENT '访问量',
  `approval_status`   ENUM('pending','approved','rejected') DEFAULT 'pending' COMMENT '审核状态',
  `publish_tag`       ENUM('system','user') DEFAULT 'system' COMMENT '发布标签',
  `user_id`           INT UNSIGNED DEFAULT NULL COMMENT '发布用户ID',
  `reject_reason`     TEXT COMMENT '拒绝原因',
  `created_at`        DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at`        DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 域名历史报价记录
CREATE TABLE `domain_offer` (
  `id`          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `domain_id`   INT UNSIGNED NOT NULL,
  `offer_price` DECIMAL(12,2) NOT NULL,
  `offered_at`  DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`domain_id`) REFERENCES `domain`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 域名访问日志
CREATE TABLE `domain_visit_log` (
  `id`          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `domain_id`   INT UNSIGNED NOT NULL,
  `ip`          VARCHAR(45) DEFAULT '',
  `user_agent`  TEXT,
  `referer`     VARCHAR(500) DEFAULT '',
  `visited_at`  DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`domain_id`) REFERENCES `domain`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 六、关键技术决策

### 6.1 LLM 调用策略
- PHP cURL 调用大模型 API（DeepSeek / 通义千问）
- 异步队列（Redis）处理，避免请求阻塞
- 结果缓存 Redis（TTL 24h）
- 降级：LLM 不可用时允许管理员手动填写

### 6.2 Whois 查询策略
- PHP 封装的 Whois 查询库
- 优先查 Redis 缓存
- 失败时允许手动填写注册时间和年限

### 6.3 第三方链接导入流程
```
输入链接 → cURL 抓取 HTML → DOM 解析提取字段
  → 字段完整? → 是 → 预览 → 确认导入
  → 否 → LLM 补充 / Whois 补充 → 预览 → 确认导入
  → 域名已存在? → 提示覆盖/跳过
```

### 6.4 权限模型 (RBAC)
- **管理组**: 所有权限
- **名商组**: 自己的域名 CRUD、导入、自己的留言管理、提交/查看需求
- 通过中间件实现接口级权限控制

### 6.5 WebSocket 实时聊天
- 基于 Workerman / Swoole 独立进程运行
- 与主 PHP 应用共享 Redis/MySQL
- 房间按会话隔离，消息持久化

## 八、开发阶段规划 (Phase 1 → 6)

| 阶段 | 内容 | 核心任务 |
|------|------|----------|
| **Phase 1** | 核心展示功能 | 首页域名卡片列表、搜索/筛选/排序、域名详情页、留言提交、委托提交、关于我们、后台登录、域名CRUD、留言管理、委托管理 |
| **Phase 2** | AI + 导入 + 可注册域名 | LLM 自动分析域名含义/行业、Whois 查询、域名导入、可注册域名管理、用户需求管理 |
| **Phase 3** | 审批流程 | 域名审批（通过/拒绝）、审核邮件通知、发布标签、用户发布页 |
| **Phase 4** | 用户系统 | 邮箱注册登录、微信扫码登录、用户角色权限（名商组）、用户管理 |
| **Phase 5** | 扩展功能 | 收藏功能、在线聊天（WebSocket）、域名估值工具、热度算法 |
| **Phase 6** | 第三方对接 | 阿里云/腾讯云/西部数码 API、Whois 历史记录、统计分析 |
